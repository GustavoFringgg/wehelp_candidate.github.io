Wehelp第五屆-秉宏申請書

兩百字內的個人簡介。
彭彭老師好，各位助教好，我叫蔡秉宏，今年即將滿33歲，畢業於世新大學資訊管理學系， 有四年航空地勤經驗，以及四年球鞋電商管理部門的經驗。

轉職的契機，是在工作中對公司的內網提出優化方案，慢慢發掘自己對程式運作邏輯的興趣，進而認識前端 / 後端 / 資料庫。更重要的是， 由於工作的環境封閉，我想跟更多厲害的人交流、學習，雖然33歲在轉職的路上有一定的風險與挑戰， 但我想給自己一個機會，給自己一個，突破自身極限的機會。


為了成為軟體工程師，做過什麼努力？若有具體作品請分享給我們。
從去年八月開始踏上轉職的道路上，我初步的方向為 1.認知前、後端分別　2.尋找網路資源　3.調整自身心態與生活習慣

起初我從Youtube、Python學習網，PTT-soft_job開始廣泛查詢轉職資訊、前後端資訊。雖然大學畢業於資訊管理，當初主要學習C++/VB ，但畢業後就沒在使用也幾乎忘光光，而轉職就是歸零的開始，我所選擇的第一個語言是Python。透過網路免費資源，我學習基本資料形態、if/elif用法、 while loop、for loop、def函數、class用法等等，讓自己逐步習慣程式語言的框架與邏輯，便開始尋找udemy課程資源，透過英文教材深耕程式邏輯概念，直至找到彭彭老師的教學影片。

雖然前面已有幾次經驗，但我再一次的重新學習，這次，我開始學習如何建立屬於自己的Notion資料庫，熟稔python的邏輯架構與語法，學習flask架設網站呼叫，學習MYSQL新增/修改/查詢語法，也因為設計WeHelp申請書，我開始學習HTML架構、CSS排版。 在未來，我更想學習github版本控制，以及能在Medium持續地撰寫自己的學習過程，在一月時也發現有Wehelp coding初學者的練習題目，也開始著手練習/解題並記錄Notion

而我自認做最大的努力，就是改變自己的作息，讓自己每天六點起床，學習兩小時，再準備上班，下班再回家繼續進修，我知道我的時間有限，在轉職的內心掙扎、焦慮，我必須要不斷調整自己的心態，竭盡所能讓自己能更像一個軟體工程師，Fake it till make it。


如果參與這個計畫，會怎麼安排學習時間？。
在去年十二月初決定要報名WeHelp培訓時，我便下定決心要全職踏入轉職的旅程，也開始安排工作預計到三月底結束，如果有幸能參與WeHelp計畫，我便會在四月全職投入轉職培訓行列，至此，我給自己訂出三個步驟。

step1.先建立心態、並維持

step2.再建立學習、再深入

step3.保持複習、再複習

首要建立心態面，在高齡轉職的心境上，出現更多的是懷疑自己的聲音，為了保持身心靈健康，我學習如何冥想，起床跟睡前各冥想10分鐘，可以讓我有精神的早起以及更快速的入眠， 我也知道必須透過運動分泌多巴胺，讓自己身心靈健康，所以我也開始慢跑放空，放空也是讓腦袋能裝下更多的知識。

建立良好的心態，接下來建立學習系統，以及有效複習。我盡可能找到WeHelp歷屆學長姐的心得分享，在文章中我發現他們都有一個共同的建議，就是「提前準備」。

有幸透過WeHelp論壇以及學長姊的分享，大概能了解整體訓練營的過程與項目，而目前的策略，採取的就是「提前」，我在今年一月初步了解第一階所需要的技術知識與目標，也計算距離第五屆開課日，還有近三個月時間能提前準備，以技術層面來說，我需要更熟稔HTML、CSS、JavaScript、Python、Flask、MySQL等相關技術， 以背景知識來說需要學習RWD 版面設計、HTTP 通訊協定、網站三層式架構(謝謝學長姐們分享)。我的預期目標，希望能在籌備期，熟悉第一階段所需要的知識，並在開始第一階段，就能開始第二階段的籌備，在第二階段，就能開始準備第三階段的構思。

目前具體規劃方式，為早上前端學習，下午後端學習，傍晚總複習與額外知識補充，執行方式也以番茄工作法去執行，25分鐘學習，5分間歇性的短休息，讓自己能維持長效性的學習，而在傍晚更為重要的，就是複習，著名的遺忘曲線(Forgetting curve)表示，有效學習必須透過階段性回想/複習，來將短期記憶轉換成長期記憶， 所以把重要的總複習，放在傍晚執行。

以上的策略是我評估"最好"的情況，在執行過程中勢必有需要調整的時候，在未來我會以"週"為單位衡量此方案的成效，並在週末檢討/思考是否能有優化或調整的地方。

轉職的目的在於能成功在就業市場生存，我篩選軟體公司的標準希望能以下述為主。
  1. 能參與跨國專案
  2. 主要業務為開發

請描述一件產生明顯負面情緒的經歷，如何處理該情緒？
在職場待了幾年，最容易產生負面情緒，就是與同事溝通產生衝突的時候，不管是處理方式的不同，或者表達錯誤造成誤會，都是造成衝突的因子，我知道在衝突且情緒高漲的情況下， 所說出的話，所做的任何決定，通常都是不理性且容易造成更大的衝突。

我自認是懂得處理自己情緒的人，當衝突發生時，第一步就是先抽離自己，要以完全客觀的角度看整件事的對錯與否，「對」就找到雙方都能接受的處理方式，「錯」，就乖乖認錯，並思考如何在下次避免這種情況再次發生。當然不是每個人天生就懂得該如何處理情緒，都是需要經過不斷地思考、練習、以及學習所積累而成。我相信自己，轉職並非完全從零開始，經過先前職場的磨練，我更能知道，該如何照顧自己的情緒。


關於這份申請網頁，分享一個開發時的技術心得。
在學習HTML與CSS時，老師給予範例成為我很好的效仿對象，讓我能有初步的picture，再從大目標拆解成小目標，逐一實行。我將所拆解的小目標區分成1.標題結構 2.內文結構 3.背景結構

1.標題方面使用border外框，將框線設定為白色，設定css width為70%，使用margin: 0 auto讓區塊置中，並google css字體使用，查詢可用font-family: fantasy更換字體。

2.內文方面，我區分"問題"以及"回答"兩種區塊，回答方面，也是使用border做區塊分割，並用text-align跟margin: 0 auto做置中，使用border-radius來讓外框邊角更圓滑。回答方面依序接著"問題"的項目做延伸， 並使用text-indent: 2em做空行的簡易排版，以及為了讓閱覽者能更聚焦在我的回答，我使用hover屬性，讓鼠標移到該區塊時，使該色區塊變色，內文所提到的關鍵字便使用span color標籤變色標示，以及為了呈現Notion筆記的足跡， 也用a href插入連結，能讓老師以及助教們參酌。

3.背景方面，我使用自己拍攝的照片，用background-image導入，background-size: cover使畫面填滿，並用repeat: no-repeat畫面不重複以及position: center讓畫面置中，也用疊圖的方式附上自己照片，並不斷調整距離，最後成為自己想呈現的樣子。

以上為關於此申請書的技術心得，由於是第一次撰寫技術心得，我甚至不知道這算不算是一種技術心得，但我覺得任何軟體的開發初衷皆以"人"為出發點，所以設計此申請書的方向時，以閱聽者閱讀舒適為主。


請用簡短兩三句話，描述對你而言，最重要的一項人際交往原則。
對我來說最重要的人際交往原則就是真誠、誠信，人是很複雜的動物，對我而言，只要彼此拿出誠意，真誠就是溝通最好的方式。


其他想要對我們說的事情？
在此，想聊聊更多自己在轉職的感受與體悟。我目前在運動球鞋電商產業工作，雖然福利不錯，但每天都在執行例行性的工作，不知道什麼時候開始，我不斷在想「自己是否就是這樣了、 我還有更多的可能嗎。」而立之年的我，從航空業到電商的經歷，沒有什麼硬實力，只有一些基礎的軟實力，隨著了解到自己的天花板日漸逼近，那種壓力跟焦慮感也日漸倍增，我想，此時如果不想辦法自己打破，也沒人會幫我打破了。

學習程式的路途上，一言以蔽之，就是學海無涯，我很幸運我還保有一定的好奇心與學習心，不會就問，不懂就查，透過不斷吸收新知識，再拓展自己的技能樹，把握住每次的機會。

很多人說，這年頭景氣不好，科技業裁員，有工作就就不錯了，也有人說，年過三十了，還是求個穩定就好，不要想東想西，更多人說，再怎麼厲害，要怎麼拚得過大學四年就在寫程式的學生，面對各種質疑自己內心的聲音，我想引用我老闆在會議上所說的一句話「你就是你自己的英雄」， 而這句話也深深影響了我，遇到挫折、遇到困境，我始終相信我自己，我就是自己的英雄。

最後，我想給自己一個機會，也謝謝老師、助教們閱讀至此，我叫蔡秉宏， 今年33歲，我還在人生的道路上奔跑。
